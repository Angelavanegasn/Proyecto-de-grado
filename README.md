Descripción general del prototipo

El prototipo desarrollado corresponde a una aplicación web orientada a la predicción y monitoreo de desastres naturales en Colombia,
mediante el uso de técnicas de inteligencia artificial y visualización de datos en tiempo real.

El sistema permite analizar información climática y generar alertas tempranas, facilitando la toma de decisiones en la gestión del riesgo.
Este prototipo cumple con un nivel de maduración tecnológica TRL5, ya que ha sido validado en un entorno simulado, demostrando su funcionalidad básica.

 Módulos del sistema

El sistema está compuesto por tres módulos principales:

<img width="961" height="71" alt="image" src="https://github.com/user-attachments/assets/90e36a38-bef5-48d1-97b8-0178bf2bc460" />


1. Mapa de alertas

Tal como se observa en la imagen 2 (página 2), este módulo presenta una representación gráfica del territorio colombiano donde se visualizan los puntos de alerta en diferentes regiones.

Cada punto se identifica mediante un código de colores que indica el nivel de riesgo:
🔴 Rojo: Nivel crítico
🟠 Naranja: Nivel alto
🟡 Amarillo: Nivel medio
🔵 Azul: Nivel bajo
🟢 Verde: Nivel seguro

Este módulo permite identificar de forma rápida las zonas con mayor riesgo, facilitando la toma de decisiones preventivas.

<img width="652" height="535" alt="image" src="https://github.com/user-attachments/assets/a0ba0794-67a5-497c-b241-051743e12e98" />

2. Sistema de alertas

El sistema incluye un módulo de gestión de alertas, en el cual el usuario puede:

Activar o desactivar el sistema de alarmas
Configurar si desea recibir alertas sonoras
Recibir notificaciones según su ubicación geográfica

Como se describe en la página 2 del documento, este módulo permite notificar al usuario sobre eventos de riesgo en su región, indicando el tipo de amenaza y su nivel de gravedad.

<img width="324" height="447" alt="image" src="https://github.com/user-attachments/assets/0f0f3b0f-e1dc-4c9b-be3a-b22b14b5233e" />

3. Generación de nuevas alertas

En la imagen 3 (página 3) se observa el formulario para la creación de nuevas alertas, el cual permite registrar eventos detectados en el sistema.

Este módulo incluye los siguientes campos:

Tipo de desastre (ej. terremoto, inundación)
Región afectada
Nivel de severidad
Tiempo de anticipación
Descripción del evento

Una vez diligenciada la información, el usuario puede generar una nueva alerta que será procesada por el sistema.
